Instalar o PyMySQL

Antes de qualquer coisa, você precisa instalar a biblioteca que faz o Python conversar com o MySQL. PyMySQL é um pacote externo — ele não vem com o Python por padrão.

O que é

PyMySQL é uma biblioteca (conjunto de código pronto) que implementa o protocolo de comunicação do MySQL. Ela traduz os comandos Python para algo que o MySQL Server entende — e vice-versa.

Onde usar

Execute esse comando uma única vez no terminal (fora do Python). Após instalado, basta importar no seu script.

terminal 
pip install pymysql
💡
Se você usa ambientes virtuais (recomendado), ative o ambiente antes de instalar. Se usar pip3 no seu sistema, substitua pip por pip3.
verificar_instalacao.py 
import pymysql
print(pymysql.version_info)  # deve mostrar a versão instalada
2

Criar a Conexão

A conexão é a "ponte" entre o seu script Python e o banco de dados. Sem ela, nenhuma operação é possível. Você cria a conexão uma vez e a usa para todas as operações.

O que é — pymysql.connect()

É uma função que abre uma conexão com o servidor MySQL. Ela recebe as credenciais e informações do banco e devolve um objeto de conexão que você guarda em uma variável.

Onde usar

No início do script, antes de qualquer consulta ou modificação. Normalmente fica logo após os imports. Você usa essa variável de conexão durante toda a execução do programa.

conectar.py 
import pymysql

# Cria a conexão com o banco de dados
conexao = pymysql.connect(
    host="localhost",        # endereço do servidor (local = "localhost")
    user="seu_usuario",     # usuário do MySQL
    password="sua_senha",   # senha do MySQL
    database="nome_do_banco",# banco que você já criou
    port=3306                 # porta padrão do MySQL (opcional)
)

print("Conexão estabelecida com sucesso!")
Parâmetro O que significa Exemplo
hostEndereço do servidor MySQL"localhost" ou "192.168.1.10"
userNome do usuário do MySQL"root"
passwordSenha do usuário"minha_senha"
databaseNome do banco de dados a usar"loja"
portPorta do servidor (padrão: 3306)3306
⚠️
Conexão aberta = recurso consumido. O MySQL tem um limite de conexões simultâneas. Sempre feche a conexão quando terminar com conexao.close(). Vamos ver uma forma elegante de fazer isso no Passo 6.
3

Criar o Cursor

A conexão abre a porta com o banco — mas quem realmente executa os comandos SQL é o cursor. Pense nele como um "ponteiro" ou "caneta" que navega e escreve dentro do banco.

O que é — conexao.cursor()

É um método do objeto de conexão. Ele cria e devolve um objeto cursor. O cursor é quem vai receber seus comandos SQL (SELECT, INSERT, etc.) e executá-los.

Onde usar

Logo após criar a conexão, antes de qualquer comando SQL. Em geral, você cria um cursor, executa as operações desejadas e depois fecha o cursor.

cursor.py 
import pymysql

conexao = pymysql.connect(
    host="localhost", user="root", password="senha", database="loja"
)

# Cria o cursor — versão simples (retorna tuplas)
cursor = conexao.cursor()

# Cria o cursor — versão dicionário (retorna {"coluna": valor})
cursor = conexao.cursor(pymysql.cursors.DictCursor)
💡
Qual cursor usar?
• Cursor simples → resultados chegam como (valor1, valor2)
• DictCursor → resultados chegam como {"nome": "João", "idade": 30} — muito mais fácil de usar!
4

Ler Dados — SELECT

Agora sim vamos buscar dados do banco. O fluxo é sempre o mesmo: execute o SQL → fetch os resultados → use os dados.

O que é — cursor.execute()

É o método que envia o comando SQL para o MySQL executar. Ele não devolve os dados — apenas executa. Para pegar os resultados, você usa os métodos fetch logo depois.

O que é — fetchall() / fetchone() / fetchmany()

São métodos para buscar os resultados após um execute():
• fetchall() → traz todos os registros de uma vez (lista)
• fetchone() → traz apenas o próximo registro (dicionário/tupla)
• fetchmany(n) → traz os próximos n registros

Onde usar

Sempre que quiser consultar dados. Use fetchall() para listagens, fetchone() quando espera um único resultado (ex: busca por ID).

select.py 
import pymysql

conexao = pymysql.connect(
    host="localhost", user="root", password="senha", database="loja"
)

cursor = conexao.cursor(pymysql.cursors.DictCursor)

# ── Buscar todos os registros ───────────────
cursor.execute("SELECT * FROM clientes")
todos = cursor.fetchall()

for cliente in todos:
    print(cliente["nome"], cliente["email"])

# ── Buscar um único registro por ID ────────
cursor.execute("SELECT * FROM clientes WHERE id = 1")
cliente = cursor.fetchone()
print(cliente)  # {'id': 1, 'nome': 'Maria', 'email': '...'}

# ── Buscar com filtro dinâmico (SEGURO) ────
nome_busca = "João"
cursor.execute("SELECT * FROM clientes WHERE nome = %s", (nome_busca,))
resultado = cursor.fetchall()

conexao.close()
🚫
NUNCA faça isso:
cursor.execute("SELECT * FROM clientes WHERE nome = '" + nome + "'")

Isso abre brecha para SQL Injection — um ataque que pode destruir seu banco. Sempre use %s com uma tupla de valores separada.

Escrever Dados — INSERT, UPDATE, DELETE

Operações que modificam o banco seguem um passo extra: o commit. Sem ele, as mudanças ficam pendentes e são descartadas quando a conexão fecha.

O que é — conexao.commit()

É o método que confirma e salva permanentemente as mudanças feitas no banco. É o equivalente a clicar em "Salvar". Deve ser chamado após todo INSERT, UPDATE ou DELETE.

O que é — conexao.rollback()

Desfaz todas as operações feitas desde o último commit. Use dentro de um bloco except para garantir que dados inválidos não sejam salvos em caso de erro.

Onde usar

Chame commit() após cada operação de escrita bem-sucedida. Chame rollback() dentro do bloco except para reverter em caso de falha.

escrita.py 
import pymysql

conexao = pymysql.connect(
    host="localhost", user="root", password="senha", database="loja"
)
cursor = conexao.cursor()

try:
    # ── INSERT: adicionar um novo registro ──────
    sql_insert = "INSERT INTO clientes (nome, email) VALUES (%s, %s)"
    cursor.execute(sql_insert, ("Ana Lima", "ana@email.com"))
    conexao.commit()  # ← confirma o INSERT
    print("Inserido com sucesso! ID:", cursor.lastrowid)

    # ── UPDATE: atualizar um registro existente ─
    sql_update = "UPDATE clientes SET email = %s WHERE id = %s"
    cursor.execute(sql_update, ("novo@email.com", 1))
    conexao.commit()  # ← confirma o UPDATE
    print("Linhas afetadas:", cursor.rowcount)

    # ── DELETE: remover um registro ─────────────
    cursor.execute("DELETE FROM clientes WHERE id = %s", (5,))
    conexao.commit()  # ← confirma o DELETE

except Exception as erro:
    conexao.rollback()  # ← desfaz tudo se algo deu errado
    print("Erro! Operação revertida:", erro)

finally:
    cursor.close()
    conexao.close()
Atributo do cursorO que retorna
cursor.lastrowidID do último registro inserido pelo INSERT
cursor.rowcountNº de linhas afetadas pelo último UPDATE ou DELETE

Boas Práticas — Código Profissional

Agora que você já sabe o básico, veja como organizar seu código para que ele seja seguro, legível e fácil de manter.

O que é — with (Context Manager)

A instrução with garante que recursos (conexão, cursor) sejam fechados automaticamente — mesmo se um erro ocorrer. Elimina a necessidade de chamar .close() manualmente.

Onde usar

Substitua o padrão manual de abrir/fechar conexão. Use with pymysql.connect(...) as conexao: e o Python cuida do fechamento por você.

profissional.py 
import pymysql

# ── Configuração centralizada ────────────────
DB_CONFIG = {
    "host": "localhost",
    "user": "root",
    "password": "senha",
    "database": "loja",
    "cursorclass": pymysql.cursors.DictCursor
}

# ── Função reutilizável para buscar clientes ─
def buscar_clientes():
    with pymysql.connect(**DB_CONFIG) as conexao:
        with conexao.cursor() as cursor:
            cursor.execute("SELECT * FROM clientes")
            return cursor.fetchall()
    # conexão e cursor fechados automaticamente aqui ✅

# ── Função reutilizável para inserir ─────────
def inserir_cliente(nome, email):
    with pymysql.connect(**DB_CONFIG) as conexao:
        with conexao.cursor() as cursor:
            sql = "INSERT INTO clientes (nome, email) VALUES (%s, %s)"
            cursor.execute(sql, (nome, email))
        conexao.commit()
        print(f"Cliente {nome} inserido!")

# ── Usando as funções ────────────────────────
clientes = buscar_clientes()
for c in clientes:
    print(c["nome"])

inserir_cliente("Carlos Silva", "carlos@email.com")
🔐
Dica extra: proteja suas credenciais.
Nunca coloque senha direto no código se for compartilhar. Use um arquivo .env:

1. Instale: pip install python-dotenv
2. Crie o arquivo .env: DB_PASSWORD=sua_senha
3. No Python: from dotenv import load_dotenv; import os; load_dotenv()
4. Use: password=os.getenv("DB_PASSWORD")
